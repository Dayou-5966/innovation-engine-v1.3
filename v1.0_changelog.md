# The Innovation Engine v1.0 Changelog

*This document outlines all the modifications, refactors, and feature additions implemented during the v1.0 development cycle.*

## 🌟 Core Features & Architecture

*   **Global Mandate Documents:** 
    *   Migrated the Mandate Documents section from the Evaluation Funnel out to the global left sidebar. 
    *   Mandate documents now act as continuous global constraints across all features.
    *   Added a scrollbar mechanism to the uploaded documents list for easier management of multiple files.
*   **Granular AI Agent Overrides (Option B):** 
    *   Added a new "⚙️ Advanced Agent Overrides" collapsible drawer inside the Evaluation Funnel.
    *   Users can now inject custom, stage-specific directives (e.g., "Exclude North America," "Ensure CAPEX < $5M") directly into the system prompts of the Strategist (Stage 1), Market Researcher (Stage 2), Quant Specialist (Stage 3), and Synthesizer (Stage 4) agents.
*   **Pipeline Cancellation & Partial Exports:** 
    *   Introduced a **Stop** button to immediately halt long-running deep evaluations.
    *   The backend now seamlessly catches cancellation signals to save LLM tokens and execution time.
    *   Halted pipelines now instantly generate a "Partial Evaluation Report," safely formatting and rendering whichever intermediate findings (Strategist Brief, Market Report, etc.) were successfully completed before the kill switch was activated.
*   **Genesis Mode Execution Logic:** 
    *   Disabled the auto-start functionality when a user selects a seed concept from the Genesis dashboard. 
    *   The idea is now pre-filled in the Evaluation Funnel, giving the user an explicit pause to construct Advanced Agent Overrides before manually hitting "Run Deep Evaluation."

## 🎨 Branding & UX Polish

*   **App Rebranding:** 
    *   Systematically updated application labels, the main dashboard header, and the browser tab string to accurately reflect the brand name: **"The Innovation Engine (TIE)"**.
    *   Ensured the browser favicon icon matches the app logo.
*   **Documentation Cleansing:** 
    *   Updated the internal "How to Use" guide text.
    *   Removed outdated references to obsolete features (such as `ngrok`) to ensure instructions accurately match the production environment.
